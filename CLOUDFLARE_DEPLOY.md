# Cloudflare Workers Deploy With OpenNext

This app is a dynamic Next.js app with API routes, middleware, Supabase Auth, and realtime staff chat. Deploy it with the OpenNext Cloudflare adapter for Workers, not static Pages drag-and-drop.

## Requirements

- Node.js 22 or newer for current Wrangler 4.
- Cloudflare account.
- Wrangler logged in locally.
- Supabase project URL and publishable anon key.

## Local Setup

Install dependencies:

```powershell
npm install
```

Log in to Cloudflare:

```powershell
npx wrangler login
```

For local Workers preview, create a local `.dev.vars` file from `.dev.vars.example`:

```powershell
Copy-Item .dev.vars.example .dev.vars
```

Fill:

```env
NEXTJS_ENV=development
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_publishable_anon_key
```

Do not put Supabase service role keys in this project.

## Cloudflare Environment Variables

Add these variables for the Worker:

```txt
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
```

Where to add them:

1. Cloudflare Dashboard.
2. Workers & Pages.
3. Select the Worker named `ace-admin-panel` after first deploy, or create it through deploy.
4. Settings.
5. Variables and Secrets.
6. Add both variables.

For CLI-based secrets:

```powershell
npx wrangler secret put NEXT_PUBLIC_SUPABASE_URL
npx wrangler secret put NEXT_PUBLIC_SUPABASE_ANON_KEY
```

Although these are `NEXT_PUBLIC_` values, storing them as Worker variables keeps the runtime environment consistent for OpenNext.

## Commands

Normal local Next.js development still works:

```powershell
npm run dev
```

Build for Cloudflare Workers:

```powershell
npm run build:cloudflare
```

Preview in the Cloudflare Workers runtime:

```powershell
npm run preview:cloudflare
```

Deploy from local CLI:

```powershell
npm run deploy:cloudflare
```

## Files Added

- `wrangler.jsonc`
- `open-next.config.ts`
- `.dev.vars.example`
- `public/_headers`

## Notes

- Do not set `output: "export"` in `next.config.ts`.
- Do not use Cloudflare Pages static drag-and-drop for this app.
- `wrangler.jsonc` uses `.open-next/worker.js` and `.open-next/assets`, which are generated by `opennextjs-cloudflare build`.
- The Worker must use `nodejs_compat`.
- If deployment fails on Node version, upgrade local Node to 22 or newer and rerun the command.
